# Context Note (맥락 노트)

## 전체 맥락
- 사용자의 운영 편의성을 위해 CMS 기능을 대폭 강화하고, 디자인의 완성도를 높임.
- 특히 이미지 업로드와 태그/카테고리 관리의 유연성을 확보함.

## 주요 결정 사항 및 시방서 목록
1. **CMS 입력 시스템 고도화 (2026-03-03)**
   - 이미지: `FileReader` API를 이용해 로컬 파일을 `Base64` 스트링으로 변환하여 `localStorage`에 저장 (이미지 서버 없이 구현 가능).
   - 태그: 1개에서 다중 배열(`tags`)로 확장. 최대 10개 이상 입력 가능하도록 UI 구성.
   - 카테고리: 기존 데이터에서 추출한 목록을 제공하되, 자유로운 신규 입력도 가능하게 함.
2. **디자인 톤앤매너 조정 (2026-03-03)**
   - `blog.html`: 매거진 스타일에서 '정사각형 섬네일 + 우측 텍스트'의 가로형 레이아웃으로 변경하여 가독성 증대.
   - 포인트 컬러: 기존 로즈 색상의 태그를 블랙/화이트 모노톤으로 변경하여 전문가적인 'Cold Logic' 분위기 강조.
3. **사용자 경험(UX) 개선 (2026-03-03)**
   - "Direct Consultation"이라는 영어 표현을 "비공개 상담 신청"이라는 더 직관적인 한국어 표현으로 치환.

## 코드 점검 사항
- 이미지 업로드 시 `Base64` 용량이 `localStorage` 한계(보통 5MB)를 넘지 않도록 안내 필요 (현재는 소형 이미지 권장).
- `view.html`과 각 리스트 페이지에서 `tags` 배열이 정상적으로 출력되는지 상호 체크 완료.
